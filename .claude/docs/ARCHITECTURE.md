## Module Responsibilities

### extension.ts
Single orchestrator. Owns:
- `activate()` / `deactivate()` lifecycle
- Command registration (`tutorcode.start`, `tutorcode.toggle`, `tutorcode.pause`, `tutorcode.revisePlan`)
- Event bus — routes monitoring events to tierEvaluator
- Cooldown gate — enforces 30s minimum between UI interruptions

### auth/claudeAuth.ts
Resolves credentials in priority order. Returns a unified `Credentials` object
consumed by the Anthropic provider. See `docs/AUTH.md`.

### auth/secretStorage.ts
Thin wrapper around `vscode.ExtensionContext.secrets`. Handles get/set/delete
for non-Anthropic provider API keys.

### state/sessionManager.ts
Reads/writes `.vscode/tutorcode.json`. Owns the current session: goal, plan,
current step, action history, provider config. Emits change events.

### state/projectIndexer.ts
Builds and maintains `.vscode/tutorcode-index.json`. Contains file tree snapshot
and per-file summaries for key files. Re-indexes on significant structural change
(file create/delete). Summaries generated by a single Haiku call, cached.

### ai/providers/anthropic.ts
Wraps Anthropic API. Two clients: Sonnet (plan/guidance/chat) and Haiku (monitoring).
Handles streaming for chat panel. Always applies prompt caching headers.

### ai/providers/openai.ts
Handles OpenAI and Grok (base URL swap). Same interface as anthropic.ts.

### ai/promptCache.ts
Builds the cached message prefix: system prompt + plan + project structure.
Marks blocks with `cache_control: { type: "ephemeral" }`. Recomputes only when
plan changes.

### ai/tierEvaluator.ts
Routes events through the three-tier system. See `docs/TOKEN_EFFICIENCY.md`.
Returns a `EvaluationResult` with `status` and optional `message`.

### monitoring/fileWatcher.ts
Subscribes to `onDidChangeTextDocument`, `onDidCreateFiles`, `onDidDeleteFiles`,
`onDidSaveTextDocument`. Debounces to 1000ms. Emits normalized `FileEvent`.

### monitoring/terminalWatcher.ts
Subscribes to `onDidWriteTerminalData`. Parses command strings from shell output.
Emits `TerminalEvent` with command string and exit code when available.

### monitoring/diagnosticsWatcher.ts
Subscribes to `onDidChangeDiagnostics`. Filters to error-level only. Emits
`DiagnosticsEvent` with file URI and error list.

### monitoring/checkpointDetector.ts
Pure logic module. Given current step definition + recent events, returns
`CheckpointResult`: `{ hit: boolean, reason: string }`. No AI calls — Tier 1 only.

### ui/guidePanel.ts
Manages the Guide Mode webview (ViewColumn.Three, ~300px wide). Sends step
content to React bundle. Receives [Got it] / [Need a hint] / [Pause] button events.

### ui/chatPanel.ts
Manages the Chat Mode webview (sidebar or editor column). Full conversation
history. Streams AI responses. Toggle button routes back to guide mode.

### ui/statusBar.ts
Bottom status bar item. Shows current step (e.g., "TutorCode: Step 3/8").
Click opens guide panel.

## Message Protocol (Webview ↔ Host)

All messages are typed. Define a shared `messages.ts` with discriminated unions:

```typescript
// Host → Webview
type HostMessage =
  | { type: 'step_update'; step: Step; totalSteps: number }
  | { type: 'character_speak'; message: string; mood: 'neutral' | 'warn' | 'praise' }
  | { type: 'chat_chunk'; text: string; done: boolean }
  | { type: 'mode_change'; mode: 'guide' | 'chat' }
  | { type: 'plan_display'; plan: Plan }

// Webview → Host
type WebviewMessage =
  | { type: 'got_it' }
  | { type: 'need_hint' }
  | { type: 'pause'; minutes: number }
  | { type: 'chat_send'; text: string }
  | { type: 'toggle_mode' }
  | { type: 'plan_suggestion'; text: string }
  | { type: 'plan_commit' }
```

## Data Flow: Monitoring Event → User Sees Message

```
1. fileWatcher emits FileEvent (debounced 1s)
2. extension.ts receives event
3. cooldown gate: has 30s passed since last interruption? if no → drop
4. tierEvaluator.evaluate(event, currentStep, projectIndex)
   a. Tier 1: local rules → if verdict clear → return immediately
   b. Tier 2: Haiku call with cached prefix + small delta → JSON verdict
   c. Tier 3: only if Tier 2 returns 'needs_guidance' or user clicked hint
5. EvaluationResult returned to extension.ts
6. if status === 'on_track' → silent, maybe status bar tick
7. if status === 'minor_drift' → character speaks (non-blocking bubble)
8. if status === 'major_drift' → character speaks + status bar warning
9. if status === 'step_complete' → character speaks + [Advance?] prompt
10. cooldown timer reset
```
